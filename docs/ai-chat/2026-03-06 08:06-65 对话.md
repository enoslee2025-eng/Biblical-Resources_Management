**时间**: 2026-03-06 10:15

**用户**: 点击自动排版后，经文还是没有按节数一行一行分开，还是一大段

**Claude**: 找到根本原因：TipTap 富文本编辑器的 StarterKit 不支持 `<sup>` 标签。当 `loadRawRichFromChapter` 生成 `<p><sup>1</sup>text</p>` 格式的 HTML 时，TipTap 无法识别 `<sup>`，导致多个 `<p>` 段落被合并成一段连续文本。

修复方案：
1. **节号改用 `<strong>` 标签**（StarterKit 原生支持），替代不支持的 `<sup>`
2. **`<p>` 标签之间添加换行符**（`.join('\n')`），帮助解析器正确识别段落边界
3. **节号与正文之间用全角空格 `\u3000` 分隔**，视觉清晰
4. **`saveRawRichToChapter` 同步更新**：移除 `<strong>` 标签和开头全角空格
5. **富文本和文本编辑两种模式都修复**：`loadRichTextFromChapter` 也改为每节一段

**修改的文件**:
- `website/src/views/BibleEditor.vue`: loadRawRichFromChapter 和 loadRichTextFromChapter 改用 `<strong>` 替代 `<sup>`，saveRawRichToChapter 更新解析逻辑
- `website/src/components/RichTextEditor.vue`: CSS 样式从 `sup` 改为 `strong`
