# 对话 99

**时间**: 2026-03-06 13:53

**用户**: 书卷选择器仍然显示"0"而不是书卷名称

**Claude**: 发现问题实际出在 BibleRead.vue（经文阅读页），而非 BibleEditor.vue。BibleRead.vue 的书卷下拉选择器使用 `book.name || book.title` 作为标签，但加载的书卷数据中缺少 `name` 和 `title` 属性，导致 Element Plus 显示原始索引值"0"。修复方法是导入共享常量 `BIBLE_BOOK_NAMES` 作为 fallback，同时修复了页面标题的同一问题。撤销了上次对 BibleEditor.vue 的不必要修改。

**修改的文件**:
- `website/src/views/BibleRead.vue`: 导入 BIBLE_BOOK_NAMES，修复书卷选择器标签和页面标题的 fallback 逻辑
- `website/src/views/BibleEditor.vue`: 撤销上次的 `:teleported="false"` 修改
