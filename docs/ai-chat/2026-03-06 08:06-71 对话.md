# 对话 71

**时间**: 2026-03-06 11:01

---

**用户**: 章节检测严重错误，第14章没有被正确识别为新章，仍然停留在13章中，导致创世记最终只有49章

**Claude**: 根本原因：`parseLines` 只靠"节号重置为1"来检测新章。当 PDF 文本出现 `...18(13章末节) 14(章号) 1-2那时...(14章首节)` 时，"14"被当作节号而非章号。

修复了两个问题：

1. **新增 `isLikelyChapterNum` 函数**：判断独立数字是否为章号
   - 核心规则：任何独立数字（非1）后面紧跟"节号1"开头的内容 → 章号
   - 辅助规则：数字比上一个节号大幅倒退（差值>2）→ 章号
   - 保护：num=1 永远不被判为章号（它是节号1），无经文时第一个数字不判为章号

2. **新增组合节号匹配**：支持 "1-2那时..." 格式（`^\d{1,3}[-–]\d{1,3}[中文...]`），PDF 中常见的合并节号

测试验证：创世记 1→13→14→15 章切换全部正确识别。正常连续经文不会误触发章检测。

**修改的文件**:
- `website/src/utils/bibleVerseParser.js`: 新增 isLikelyChapterNum 章号检测、组合节号匹配、parseLines 重写章节边界逻辑
