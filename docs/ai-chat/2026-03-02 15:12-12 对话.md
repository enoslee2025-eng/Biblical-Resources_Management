# 对话 12

**时间**: 2026-03-02 17:14

**用户**: 网站使用的时候不明原因会卡顿；导入整本圣经时，没有按照我希望的方式导入

**Claude**: 找到了两个问题的根源并修复：

### 问题1：网站卡顿
**原因**：每次打字时都会对整本圣经的数据做 `JSON.stringify` 来检查是否有修改，数据量大时非常耗性能。同时首页加载进度是逐个串行请求 API，资源多时会卡住界面。

**修复**：
- 脏检查增加 500ms 防抖，打字时不再每次都序列化整个圣经数据
- 首页进度加载改为并行请求（`Promise.allSettled`），所有资源同时加载

### 问题2：整本圣经导入章节分割错误
**原因**：之前用简单的正则（按空行分割）来分章节，完全无法正确识别章节边界。

**修复**：新增 `splitIntoChapters` 智能分章函数，按优先级检测：
1. **章:节格式** — 如 "1:1 起初…", "2:1 耶和华…"，按章号自动分组
2. **中文章节标题** — 如 "第一章"、"第1章"
3. **英文章节标题** — 如 "Chapter 1"、"CHAPTER 1"
4. **兜底** — 整段文本作为一章

**修改的文件**:
- `website/src/utils/fileImport.js`: 新增 `splitIntoChapters` 智能分章函数
- `website/src/views/BibleEditor.vue`: 脏检查防抖 + 用 `splitIntoChapters` 替换全部 5 处破损正则
- `website/src/views/Home.vue`: 进度加载从串行改为并行
