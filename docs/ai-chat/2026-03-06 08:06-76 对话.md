# 对话 76

**时间**: 2026-03-06 11:27

---

**用户**: 修复 BibleVerseParser 严重问题：注释/词义解释/交叉引用内容（如 `31:47"伊迦尔·撒哈杜他"意思是"做见证的石堆"。`）被错误混入了经文正文 verse.text。要求新增 Annotation / Gloss Detection 和 Canonical Text Isolation。

**Claude**: 在 BibleVerseParser 中新增了 Annotation / Gloss 检测与过滤系统：

1. **新增正则常量**：
   - `GLOSS_NOTE_FULL_RE`：匹配完整 gloss note（`31:47"术语"意思是"解释"。`）
   - `GLOSS_NOTE_SIMPLE_RE`：匹配简化 gloss note（`31:47"术语"意思是纯文本。`）
   - `REF_EXPLAIN_RE`：匹配引用+解释（`31:47意思是"解释"。`）
   - 关键词集：意思是、原意是、原意、意思为、即、解释为、就是、亦即、又名、或作、或译

2. **新增函数**：
   - `isAnnotationLine(line)`：检测整行是否为纯注释行
   - `stripAnnotationContent(text)`：从经文文本中移除内嵌的注释块
   - 4 种清理模式：完整 gloss、简化 gloss、引用解释、尾部孤立引用

3. **集成到解析流程**：
   - `filterNonScriptureLines` 新增 `isAnnotationLine` 过滤（行级）
   - `cleanVerseText` 新增 `stripAnnotationContent` 调用（内嵌级）

测试验证：
- `31:47"伊迦尔·撒哈杜他"意思是"做见证的石堆"。` → 完整移除
- `1:1"太初"原意是"在最开始的时候"。` → 完整移除
- 独立注释行 → 整行过滤
- 正常经文（含引号对话）→ 无误伤

**修改的文件**:
- `website/src/utils/bibleVerseParser.js`: 新增 Annotation/Gloss 检测系统（正则、isAnnotationLine、stripAnnotationContent），集成到 filterNonScriptureLines 和 cleanVerseText
