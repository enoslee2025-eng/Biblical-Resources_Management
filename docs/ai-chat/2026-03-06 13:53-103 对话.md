# 对话 103

**时间**: 2026-03-06 14:21

---

**用户**: 智能排版仍然将多节经文合并为一节，没有正确分割。需要让每节经文独立分段。

**Claude**: 问题根因是预处理正则表达式过于狭窄，要求节号前后必须紧跟特定字符，无法匹配带空格、全角空格等变体模式。

修复方法：
1. 将单个预处理正则改为三轮替换，逐步覆盖不同的嵌入节号模式：
   - 第一轮：句末标点（。！？等）+ 可选空格 + 数字 + 可选空格 + 中文/引号
   - 第二轮：中文字符 + 空格 + 数字 + 可选空格 + 中文/引号（无标点衔接）
   - 第三轮：中文/标点紧跟数字+中文（无空格，原有模式的扩展版）
2. 解析器 `parseStructuredChapters` 的 inline match 也扩展为支持节号与中文之间的空格
3. 解析器向前查找时的 break 条件同样扩展空格匹配

**修改的文件**:
- `website/src/views/BibleEditor.vue`: 将预处理从单正则改为三轮替换，覆盖更多嵌入节号模式
- `website/src/utils/bibleVerseParser.js`: 扩展 inlineMatch 和 lookahead 正则，支持节号后带空格
