**用户**: 第一章和第二章内容一样；民数记加载进来章节有问题

**Claude**: 发现两个问题：

1. **数据已被旧 bug 污染**：之前 v-model 竞态条件在切换章节时把旧章节内容覆盖到新章节，导致各章内容相同。代码已修复，但已存储的数据需要重新导入。

2. **导入函数解析不够智能**：`confirmImport` 和 `confirmFullBibleImport` 使用简单的 `split('\n')` + `stripLineNumbers` 来解析经文，不支持 PDF 独立行编号、多行合并、标题过滤等高级格式。改为统一使用 `splitTextToVerses`（来自 fileImport.js），该函数支持：
   - 独立行编号（PDF 格式）
   - 行首带编号前缀（1. 2. 等）
   - 按行分节 + 独立数字合并
   - 标题自动过滤
   - 按句号兜底分节

3. **删除了本地重复的 `splitTextToVerses` 函数**，改用从 fileImport.js 导入的更强大版本

4. **confirmImport 中 `saveCurrentVerses()` 缺少模式检查**：在内联模式下会用陈旧 versesText 覆盖数据

**修改的文件**:
- `website/src/views/BibleEditor.vue`:
  - 新增 import `splitTextToVerses`
  - `confirmImport`：章节/书卷导入均改用 `splitTextToVerses`；`saveCurrentVerses()` 添加模式检查
  - `confirmFullBibleImport`：改用 `splitTextToVerses`
  - 删除本地 `splitTextToVerses` 函数（改用 fileImport.js 版本）
