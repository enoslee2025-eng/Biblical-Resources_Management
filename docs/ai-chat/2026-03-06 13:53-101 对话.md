# 对话 101

**时间**: 2026-03-06 14:10

---

**用户**: 修复 Verse Number Leakage（节号残留/重复渲染）问题。自动排版后，verse.text 开头残留了原始节号（如 verse 3 的 text 是 "2神看光是好的..."），导致 UI 左侧显示节号 + 正文又包含节号。要求实现 Verse Token Stripping、stripResidualVerseNumber()、validateVerseText() 等函数。

**Claude**:

### 问题分析

两个根本原因：

1. **`autoFormatChapter` rawText 构造错误**：函数先 strip 再 prepend 入口的 verse number，但当 entry.verse（如2）与文本内嵌的实际节号（如4）不匹配时，strip 失败，prepend 产生 "24神看光..." 这样的垃圾文本。

2. **Parser 输出未清理残留节号**：parseBibleText 在 Verse Segmentation 后没有执行 Verse Token Stripping，可能在 verse.text 中留下相邻节号。

### 修复内容

**BibleEditor.vue**:
- `autoFormatChapter`: 简化 rawText 构造，直接使用 verse.text 原文（已含节号），不做 strip+prepend，避免节号不匹配时的文本损坏
- `stripVerseNum`: 增强为支持 Residual Verse Token 检测，先精确匹配当前节号，再 fallback 检测任何开头数字+中文的模式

**bibleVerseParser.js**:
- 新增 `stripResidualVerseNumber(text, currentVerse, previousVerse)`: 检测并移除经文开头的残留节号（支持当前/相邻节号匹配 + 合理范围检测）
- 新增 `validateVerseText(verses)`: 校验 verse.text 是否仍有残留节号，返回警告列表
- 新增 `stripAllResidualVerseNumbers(verses)`: 批量清理
- 在 `parseBibleText` 主函数中集成 Step 9.5: Verse Token Stripping，确保输出的 Canonical Verse Text 不含节号

**修改的文件**:
- `website/src/views/BibleEditor.vue`: 修复 autoFormatChapter rawText 构造 + 增强 stripVerseNum
- `website/src/utils/bibleVerseParser.js`: 新增 Verse Token Stripping 步骤（Step 9.5）
