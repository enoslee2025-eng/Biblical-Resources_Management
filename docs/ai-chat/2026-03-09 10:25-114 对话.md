# 对话 114

**时间**: 2026-03-09 10:25

---

**用户**: 阅览页面滚动到第4章后，点击「下一章」跳回第2章而不是第5章。

**Claude**: `prevChapter()` 和 `nextChapter()` 基于 `currentChapterIndex` 计算下一章，但滚动时只更新了 `scrolledChapterOffset`（用于显示），从未同步回 `currentChapterIndex`。用户滚动到第4章时 `currentChapterIndex` 仍然是 0（第1章），所以 +1 = 第2章。

修复：改为基于 `displayChapterIndex`（当前可见章）计算导航目标，而非 `currentChapterIndex`。

**修改的文件**:
- `website/src/views/BibleRead.vue`: `prevChapter()` 和 `nextChapter()` 基于 `displayChapterIndex` 导航
